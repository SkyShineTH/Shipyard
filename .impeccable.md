# Impeccable — Shipyard

Persistent design and UX context for this repository. Update this file when brand or audience decisions change.

## Design Context

### Users

- **Primary:** Hiring managers, interviewers, and senior engineers evaluating a **junior DevOps / platform** portfolio (per [CONTEXT.md](CONTEXT.md): new graduate with internship experience).
- **Secondary:** Peers, open-source visitors, and the owner revisiting the project for demos (local Docker, kind + Argo CD, future DOKS).
- **Context of use:** Short sessions — skim the landing, click through auth + todos, maybe check repo structure and GitOps assets. Goal is **credibility** (“this person ships end-to-end”) not daily productivity software.
- **Jobs to be done:** Understand stack at a glance → trust quality → open code or live demo → remember the project.

### Brand Personality

- **Three words:** *Credible · Technical · Calm*
- **Voice:** Confident and precise, not hypey. Prefer concrete stack terms (React, Go, Helm, Argo CD) over vague “enterprise” claims. Marketing copy must stay **honest** vs actual code (e.g. REST where that’s what ships; observability framed as *ready to add* if not wired yet).
- **Emotional goals:** Trust, clarity, quiet competence — the interface should feel **intentional** and **portfolio-grade**, not a generic template dump.

### Aesthetic Direction

- **Overall:** Modern **2025–2026** web craft: generous spacing, sharp type hierarchy, subtle depth. **Maritime / charting** metaphor is **light touch** (compass, grid, waves in parallax) — not literal branding overload.
- **Home (`/`):** Same chrome and tokens as the rest of the app: **card + `card--panel`** feature blocks, **primary/secondary** buttons, **page eyebrow** rhythm, **Material Symbols** + **scroll reveals**. No separate “dark landing” theme — **light/dark** follows `prefers-color-scheme` everywhere.
- **App chrome (global):** **System-friendly** palette via CSS variables in [services/frontend/src/index.css](services/frontend/src/index.css): cool neutrals, **teal-blue accent** (`#0d6e9e` light / `#4db3e0` dark). Subtle teal radial atmosphere + (in dark) **frosted top bar** (`topbar--workspace`).
- **Motion:** **Subtle mouse parallax** on decorative layers; **respect `prefers-reduced-motion`** (parallax and reveals degrade to static). No gratuitous animation on forms or data paths.
- **References (implicit from implementation):** Awwwards/Dribbble-adjacent **restraint** — polish without neon overload or heavy skeuomorphism.
- **Anti-patterns to avoid:** Loud purple gradients, cluttered dashboards, placeholder-lorem tone, claiming features the repo does not implement.

### Design Principles

1. **Truthful surface** — UI copy and “capabilities” align with what’s in the repo; aspirational roadmap is labeled as such.
2. **Depth without noise** — Parallax, glass, and scroll effects support hierarchy; they never block reading or interaction.
3. **Token-first styling** — Extend `:root` variables and shared classes in `index.css` before adding one-off styles; keep marketing sections scoped under `.landing-*` but reuse `card--panel`, `.btn`, and `var(--*)` like other routes.
4. **Accessible defaults** — Visible focus states, semantic headings, `aria-hidden` on decorative layers, `sr-only` for screen-reader-only titles; formal WCAG target level TBD with stakeholder (see open questions below).
5. **One accent family** — Teal/cyan carries brand and interactive emphasis; red reserved for errors/destructive actions.
